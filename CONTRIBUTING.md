# Contributing to FlowPilot

Thank you for your interest in contributing to FlowPilot! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/flowpilot.git`
3. Install dependencies: `npm install`
4. Start the dev server: `npm run dev`
5. Create a branch: `git checkout -b feature/your-feature-name`

## Development Workflow

- Write clear, descriptive commit messages
- Keep pull requests focused on a single change
- Follow the existing code style and conventions
- Test your changes locally before submitting

## Code Style

- Use TypeScript strict mode
- Follow the existing folder structure
- Use functional components with hooks
- Prefer named exports for components
- Use Tailwind CSS for styling
- Keep components small and focused

## Pull Request Process

1. Update the README if your change adds new features
2. Ensure the build passes: `npm run build`
3. Fill out the pull request template completely
4. Link any related issues
5. Request review from maintainers

## Reporting Bugs

Use the bug report template and include:

- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots if applicable
- Browser and OS information

## Feature Requests

Use the feature request template and describe:

- The problem you're trying to solve
- Your proposed solution
- Alternatives you've considered

## Code of Conduct

Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md).

## Questions?

Open a discussion or reach out to the maintainers. We're happy to help!
